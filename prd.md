产品需求文档 (PRD)：个人品牌网站 - Vibe Coding & Marathoner
1. 项目愿景 (Project Vision)
构建一个纯粹的个人展示空间，将 Vibe Coding（感性编程） 的瞬时爆发力与 马拉松 的持久忍耐力相结合。不谈职业背景，只谈“创造”与“节奏”，打造一个具有建筑审美感且极致简洁的数字名片。

2. 设计规范 (Design Specifications)
2.1 视觉调性
主题：纯净浅色模式 (Light Mode Only)。
背景色：#F8F9FA (极简灰白) 或 #FFFFFF (纯白)。
文字色：#1A1A1A (深灰黑，非纯黑)。
强调色：克制的莫兰迪色系（如 #4A90E2 或淡橙色），仅用于悬停与点缀。
布局：大留白、单列或 Bento Grid 布局，强调呼吸感。

2.2 字体排版
标题：使用具有现代感的粗体无衬线字体（如 Inter 或 Space Grotesk）。
正文：高可读性的无衬线字体，行间距设为 1.75 以确保阅读体验。

3. 页面功能模块细化
3.1 首页 (Hero Section)
核心元素：
主标题：一行极简标语，如 “Flow, Speed, and Vibes.”
简介：一段去技术化的自我定义。例如：“在心流中构建工具，在赛道上寻找节奏。专注于 AI 协作的灵感瞬间与 42.195 公里的持久意志。”
头像：侧重生活化或赛道瞬间的极简头像。
布局：移动端垂直堆叠，桌面端黄金比例分割。

3.2 关于我 (The Runner's Logic)
核心内容：
Vibe Coding 哲学：描述如何利用 AI 和直觉快速将创意转化为现实，追求“即兴构建”的快乐。
运动里程碑：
全马 PB：3:00（强调极限自律）。
半马 PB：1:24（强调速度爆发）。
共同点：探讨长跑中的“隔离感”与编程中的“心流状态”如何互补。

展示要求：不提及具体职级和公司，只展示作为“创造者”和“跑者”的身份。

3.3 项目展示 (Project Log)
展示风格：Bento Grid (便当盒) 布局，不同规模的项目占用不同大小的网格。
重点项目示例：
PB日历 (PB Calendar)：为跑者设计的训练与进度追踪工具。

展示项详情：
项目缩略图（极简 UI Mockup）。
项目名称与一句短评。
标签：#Vibe-Coding #AI-Assisted #Automation。

3.4 联系方式 (Contact)
内容：
GitHub（展示 Vibe Coding 相关的开源实验）。
Email 地址。
社交媒体链接（如 X 或微信预览）。
交互：点击邮箱自动复制或拉起客户端。

4. 交互与体验要求 (UX/UI)
移动端优先 (Mobile First)：
所有点击区域符合人体工程学。
图片在移动端自动缩放并保持清晰。

呼吸感动效：
页面加载时，元素按顺序淡入（Fade-in）。
滚动时，背景色或边框有微妙的色差反馈。

极简导航：
仅保留最必要的链接，移动端使用隐藏式抽屉菜单或底部固定栏。

5. 建议技术栈 (Tech Recommendation)
框架：Astro (适合内容驱动的个人网站，加载极快)。
样式：Tailwind CSS (用于快速实现个性化、响应式的极简 UI)。
部署：Vercel / Netlify (符合 Vibe Coding 快速上线的精神)。
动效：Framer Motion (提供丝滑的物理感交互)。

6. 后续规划 (Roadmap)
Phase 1：视觉雏形搭建，上线首页与关于我。
Phase 2：集成 Markdown/MDX，方便通过 AI 快速生成和更新项目案例。
Phase 3：加入一个小巧的“马拉松备赛倒计时”组件。
